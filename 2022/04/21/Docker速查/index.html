Docker速查

Docker

Docker安装

卸载旧版本

yum remove docker \
        docker-client \
        docker-client-latest \
        docker-common \
        docker-latest \
        docker-latest-logrotate \
        docker-logrotate \
        docker-engine

设置仓库 安装所需的软件包

yum install -y yum-utils \
device-mapper-persistent-data \
lvm2

设置稳定的仓库。

yum-config-manager \
 --add-repo \
 https://download.docker.com/linux/centos/docker-ce.repo

安装 Docker Engine-Community

yum install docker-ce docker-ce-cli containerd.io

启动 Docker

systemctl start docker

Docker自启动

systemctl enable docker.service

通过运行 hello-world 映像来验证是否正确安装了 Docker Engine-Community

docker run hello-world

获取 container ID 或者名字

docker container ls -a

如果你要删除的 container 还是运行状态，那么就要先把容器停止了：

docker  container  stop   CONTAINER_ID

删除指定的container

# 这两条命令都是删除同一个容器
docker   container  rm  CONTAINER_ID  或者 
docker  container  rm  CONTAINER_NAME

批量获取容器ID

docker container ls -a -q

批量获取镜像ID

docker image ls -a -q

批量停止容器

docker container   stop   (docker  container  ls   -a  -q)

批量删除容器

docker   container   rm  (docker  container  ls   -a  -q)

批量删除镜像

docker  image   rm   (docker  image  ls   -a  -q)

通过image的id来指定删除镜像

 docker rmi <image id>

# 想要删除untagged images，也就是那些id为<None>的image的话可以用
 docker rmi (docker images | grep "^<none>" | awk "{print 3}")

# 要删除全部images
 docker rmi (docker images -q)

dnf安装 Doceker

安装软件库

dnf config-manager --add-repo=https://download.docker.com/linux/centos/docker-ce.repo

安装 docker

dnf install -y docker-ce

启动 docker

systemctl enable docker
systemctl start docker

卸载 docker

dnf -y remove docker-engine.x86_64

rm -rf /var/lib/docker

Docker镜像加速下载

阿里云–容器镜像服务–镜像加速器

sudo mkdir -p /etc/docker 
sudo tee /etc/docker/daemon.json <<-'EOF' 
{
	"registry-mirrors": [
	    "https://wbktaw0y.mirror.aliyuncs.com",
	    "https://registry.docker-cn.com"
	]
} 
EOF
sudo systemctl daemon-reload 
sudo systemctl restart docker

Docker 搜索

按名称搜索镜像(多数这种)

docker search xxx

按名称搜索镜像并显示非截断描述

docker search --no-trunc xxx

按名称“xxx”搜索出星数至少为3颗星的镜像

docker search --filter stars=3 xxx

显示名称中包含“xxx”的镜像，并且是自动构建

docker search --filter is-automated xxx

显示的图像名称包含“xxx”，至少3颗星，并且是官方版本

docker search --filter "is-official=true" --filter "stars=3" xxx

格式化选项（–format）使用Go模板漂亮地打印搜索输出。

# 1.使用不带标头的模板,Name并StarCount为所有图像输出 以冒号分隔的条目和条目：
docker search --format "{{.Name}}:{{.StarCount}}" xxx
# 2.输出表格格式:
docker search --format "table {{.Name}}\t{{.IsAutomated}}\t{{.IsOfficial}}" xxx

docker run -di --name nginx -p 80:80 nginx

docker cp nginx:/etc/nginx /data/nginx/conf

终止并删除容器

docker stop nginx;docker rm nginx

创建 Nginx 容器

docker run -di --name nginx -p 80:80 -v /data/nginx/conf:/etc/nginx nginx

MySQL

拉取镜像

docker pull mysql:8.0.21

创建容器

docker run -di --name mysql8 -p 3306:3306 \
-v /data/mysql/conf:/etc/mysql/conf.d \
-v /data/mysql/logs:/logs \
-v /data/mysql/data:/var/lib/mysql \
-e MYSQL_ROOT_PASSWORD=root mysql:8.0.21

-e：代表添加环境变量 MYSQL_ROOT_PASSWORD 是 root 用户的登陆密码。

进入容器并使用 MySQL 命令打开客户端。

# 进入容器
 docker exec -it mysql8 /bin/bash
# 使用 MySQL 命令打开客户端
 mysql -uroot -proot --default-character-set=utf8

Redis

拉取镜像

不指定版本,则拉取最新版本的镜像
docker pull redis
指定版本
docker pull redis:5.0.5

查看拉取成功的镜像

docker images

创建容器

docker run --name redis -p 6379:6379 --restart=always -v /data/redis:/data/  -d redis redis-server --appendonly yes
# 本地运行
  -d
# 本地端口:Docker端口
  6379:6379
# 指定驱动盘
  -v
# Redis的持久化文件存储
  PWD/data
# docker的镜像名
  redis
# redis服务器
  redis-server
# 开启持久化
  --appendonly yes
# 这个运行的镜像的名称
  --name
# Docker启动容器就启动
  --restart=always
# 密码
  --requirepass "密码"

停止正在运行的镜像(redis为前面设置的镜像名称)

docker  container  stop   redis

删除镜像

docker rm redis

重启镜像

docker start redis

访问容器

docker exec -it redis bash

使用redis-cli访问容器内redis

docker exec -it redis redis-cli

RabbitMQ

拉取镜像(带管理页面版，省去了许多配置)

docker pull rabbitmq:3.9-management

创建容器

 docker run -p 5671:5617 -p 5672:5672 -p 4369:4369 -p 15671:15671 -p 15672:15672 -p 25672:25672 \
-e RABBITMQ_DEFAULT_USER=root -e RABBITMQ_DEFAULT_PASS=root \
-d  --name rabbitmq  rabbitmq:3.9-management

进入容器并开启管理功能

# 进入容器
 docker exec -it rabbitmq /bin/bash
# 如果没开启，则开启 RabbitMQ 管理功能
 rabbitmq-plugins enable rabbitmq_management

3.9.x延迟队列插件下载

mkdir -p  /docker/rabbitmq/plugins/

cd  /docker/rabbitmq/plugins/

wget https://github.com/rabbitmq/rabbitmq-delayed-message-exchange/releases/download/3.9.0/rabbitmq_delayed_message_exchange-3.9.0.ez

docker cp /docker/rabbitmq/plugins/rabbitmq_delayed_message_exchange-3.9.0.ez rabbitmq:/plugins

docker exec -it rabbitmq /bin/bash

cd /plugins

rabbitmq-plugins enable rabbitmq_delayed_message_exchange

新添加一个账户

默认的guest 账户有访问限制，默认只能通过本地网络(如 localhost) 访问，远程网络访问受限，所以在使用时我们一般另外添加用户，例如我们添加一个root用户：

进入到rabbitMq容器内部，添加用户，用户名为root,密码为root
/# docker exec -i -t rabbitmq bin/shell
/# rabbitmqctl add_user root root
赋予root用户所有权限
/# rabbitmqctl set_permissions -p / root ".*" ".*" ".*"
赋予root用户administrator角色
/# rabbitmqctl set_user_tags root administrator
查看所有用户即可看到root用户已经添加成功
/# rabbitmqctl list_users
Listing users ...
user	tags
guest	[administrator]
root	[administrator]

执行exit命令，从容器内部退出即可。

Elasticsearch

拉取镜像

docker pull elasticsearch:6.5.3

创建容器

 docker run -di --name es \
-p 9200:9200 -p 9300:9300 \
-e "discovery.type=single-node" -e ES_JAVA_OPTS="-Xms512m -Xmx2048m" \
-v /data/es:/root/ \
elasticsearch:6.5.3

进入容器

docker exec -it es /bin/bash

安装 ik 分词器：

./bin/elasticsearch-plugin install https://github.com/medcl/elasticsearch-analysis-ik/releases/download/v6.5.3/elasticsearch-analysis-ik-6.5.3.zip

修改 es 配置文件：

vi ./config/elasticsearch.yml
cluster.name: "docker-cluster"
network.host: 0.0.0.0

# minimum_master_nodes need to be explicitly set when bound on a public IP
# set to 1 to allow single node clusters
# Details: https://github.com/elastic/elasticsearch/pull/17288
discovery.zen.minimum_master_nodes: 1

# just for elasticsearch-head plugin
http.cors.enabled: true
http.cors.allow-origin: "*"

退出容器

/# exit

重启容器：

docker restart es

Oracle

拉取

docker pull alexeiled/docker-oracle-xe-11g

运行

docker run -d  --name oracle --shm-size=1g -p 1521:1521 alexeiled/docker-oracle-xe-11g

进入容器

docker exec -it  oracle /bin/bash

登录oracle

sqlplus system/oracle

创建用户并分配权限

create user 用户名 identified by 密码;

给用户deque赋予connect,resource角色，这俩角色含有许多权限.

connect:是保证该用户能连接数据库;
resource:是该用户可以使用数据库资源。

grant connect,resource to 用户名;

使用navicat创建连接。

hostname: localhost
port: 1521
sid: xe
username: 用户名
password: 密码
最初system和sys的默认密码都是oracle
